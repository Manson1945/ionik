Configuración de Bot AI.

Escrito por ionikSend, el día 12 de Octubre 2024.

Actualizado el 12 de Noviembre 2024

1. Con el administrador de cuenta ingrese en el menú lateral en la opción "WhatsApp Chatbot".

2. Haga click en el ícono del lápiz para administrar la cuenta que quiere editar.

3. Haga Click en "Bot AI".

4. Por defecto el Bot AI se encuentra desactivado, haga Click en la casilla "ACTIVO" para activarlo.

5. A continuación encontrará la configuración general: OPEN AI API KEY será la clave privada que autentica tu cuenta de OpenAI para realizar las solicitudes de IA. Sin esta clave, no se pueden generar respuestas. MOTOR AI es el modelo de lenguaje utilizado (por ejemplo, gpt-4o-mini-2024-07-18). Define la capacidad, velocidad y costo del procesamiento. FINE TUNING ID es el identificador de un modelo personalizado (si se usa fine-tuning). Dejar vacío si se utiliza un modelo base de OpenAI.

Una vez realizadala configuración general, deberá realizar la configuración de los parámetros del modelo de Bot: TEMPERATURE controla la creatividad de las respuestas. con valores bajos (0.1 0.3) obtendrá respuestas más precisas y predecibles. Con valores altos (0.7 1.0) obtendrá respuestas más variadas y creativas. MAX TOKENS es el número máximo de tokens (palabras + fragmentos) que puede generar el modelo por respuesta. Recomendado dejar en 300. TOP_P es el filtro de probabilidad acumulada. Valores cercanos a 1 permiten más diversidad. Valores bajos hacen las respuestas más restrictivas. N es el número de respuestas que el modelo debe generar por cada solicitud. Generalmente se usa 1.

7. DETECTAR NÚMERO PARA BÚSQUEDA DE DATOS Permite definir si el bot debe buscar automáticamente mediante el número WhatsApp consultas mediante API los datos del usuario. Complementa con URL ENDPOINT (FORMATO JSON). LÍMITE DE HISTORIA (MEMORIA IA) Cantidad de mensajes previos que la IA recordará en cada conversación. Controla el contexto disponible. LÍMITE DE CONSULTAS PARA EL USUARIO Número máximo de interacciones que un usuario puede realizar con este bot. 0= sin límite. TIEMPO DE ESPERA Tiempo que debe pasar antes de que el usuario pueda volver a interactuar (útil para bots con ventanas de tiempo).

8. TOTAL REQUEST Cantidad total de solicitudes enviadas a la API de OpenAI. TOKENS ENVIADOS Número total de tokens usados en los mensajes de entrada (prompts). TOKENS RECIBIDOS Número total de tokens generados por las respuestas de la IA. TOKENS TOTALES Suma de tokens enviados y recibidos. Útil para calcular el consumo total del modelo.

9. SYSTEM PROMPT Texto base que define la personalidad, tono y comportamiento del chatbot. Ejemplo: instrucciones para pedir cédula, validar datos o responder amablemente.

10. URL ENDPOINT (FORMATO JSON) Dirección del endpoint externo donde el bot consulta información (por ejemplo, productos, deudas, datos de cliente, etc.) en formato JSON. Se puede incluir una variable dinámica como {from_number},{number}, {phone}, {dni}, {documento}, {doc}, {id}, {cedula}. CARGUE EL ARCHIVO CON INFORMACIÓN LIMITADA (CSV O JSON) Permite subir un archivo con datos locales (por ejemplo, productos, deudas, datos de cliente, etc.) que la IA puede consultar sin depender de un endpoint externo.
